# 18 — 字型現代化規劃:UTF-8 + TTF 渲染

> 規劃把原版的「自製 16×16 點陣字型 + glyph 索引」文字系統,在重製版改成 **UTF-8 文字 + TTF 字型渲染**。
> 這篇是**計畫/規格**(尚未實作)。基礎見 `08`(字型/文本)、`09`(劇情)、`14`(控制碼)。

## 目標與動機

原版受限於「1824 字模、16×16 點陣、glyph 索引」(見 `08`)。重製版改用 **TTF + UTF-8** 可以:
- **不再受 1824 字限制**——任何中文字都能顯示,利於校訂、翻譯、加新劇本(呼應 `17`)。
- **高解析、可縮放**——TTF 在任何畫布大小都清晰,不像點陣放大會鋸齒。
- **可換字型風格**——明體 / 黑體 / 復古點陣 TTF 任選,甚至多語系。
- **文字資料化**——劇本變成可讀可編輯的 UTF-8,接上 `17` 的資料驅動擴充。

> 註:重製是**乾淨新引擎**(SDL2 / Ebiten),不是 patch 原 EXE,故可直接用 TTF,無須像「patch 原引擎」那樣
> 烘點陣 atlas。但仍提供「原版 16×16 點陣」模式供懷舊(見下「雙字型模式」)。

## 我們已握有的橋樑

- **`docs/data/glyph_map.json`**:glyph 索引 → Unicode(1824/1824,已校正)。
- **`docs/data/unicode_to_glyph.json`** + `tools/encode_text.py`:UTF-8 → glyph(round-trip 100%),供回寫原版。
- **`tools/decode_story_text.py`**:FDTXT → 含說話者的 UTF-8。
→ **「原版文字 ↔ UTF-8」雙向已通**,字型現代化的資料層已就緒。

## 管線:文字資料化(build-time)

把原版 FDTXT 一次轉成重製用的**結構化 UTF-8 腳本**(不在執行期查 glyph 表):

```
FDTXT.DAT ─(glyph_map)→ 每章 script.json:
  [ {box: "portrait"|"top"|"bottom", speaker: "索爾"/null,    # 開框碼 0xFFEF/EE/ED/EC
     lines: ["第一行","第二行","第三行"],                      # 0xFFFE 軟換行
     wait: true/false},                                       # 0xFFFD 翻頁等鍵
    ... ]
```
- 控制碼(`14`)→ 轉成腳本的結構欄位(開框/頭像/換行/翻頁),不再是 inline byte。
- 說話者肖像 ID → 角色名(顯示)+ 頭像資源編號(DATO,見 `01`§7)。
- 工具:擴充 `decode_story_text.py` 多一個 `--script-json` 輸出模式即可。

## TTF 渲染(runtime)

| 引擎 | TTF 方案 |
|---|---|
| **SDL2 / C++** | `SDL_ttf`(`TTF_RenderUTF8_Blended`);或 `stb_truetype` 自繪到 texture |
| **Go / Ebiten** | `text/v2` + `opentype`(`golang.org/x/image/font`)直接畫 UTF-8 |

文字繪製流程:讀 script.json → 開對話框(必要時從 DATO 載頭像)→ 逐行 TTF render → `wait` 處等按鍵翻頁。
對話框幾何沿用 `14`(每框 3 行、上/下框位置),但行寬改由 TTF 字寬決定(見排版)。

## 畫布與排版

- **畫布解析**:原版 320×200、文字 16×16 全形(每行 ~20 字)。重製拉高內部畫布(建議 **640×400 整數 2×**
  或 4:3 的 640×480),底圖 nearest 放大保持銳利(CJK hi-res 原則:拉畫布、別縮字)。
- **行寬 / 斷行**:
  - **方案 A(保留原排版)**:沿用原始 `0xFFFE` 換行點,維持「每框 3 行」的原汁版面;TTF 用接近全形的等寬字距。
  - **方案 B(重新流排)**:忽略原換行,依框寬自動 word-wrap;適合換長字型 / 翻譯後長度變動。
  - 建議:預設 A(忠於原版),翻譯 / 擴充模式用 B。
- **全形 / 半形**:漢字與標點(`『』「」，。！？`)走全形;ASCII、機器人代號(`73`/`C2`/`A1`)、純數字字模
  (`11`/`22`)走半形——這些在 glyph_map 已是對應字串,TTF 直接畫即可。

## 雙字型模式(與音源「MT-32/SoundFont」對稱)

| 模式 | 做法 | 觀感 |
|---|---|---|
| **原版點陣**(懷舊) | 把 FDOTHER#4 烘成 16×16 atlas,用 glyph 索引畫(`tools/decode_text.py` 已能匯出) | 100% 還原 1995 畫面 |
| **TTF 現代**(預設) | UTF-8 + TTF render | 清晰、可縮放、可翻譯 |
| **TTF 復古**(折衷) | 用點陣風 TTF(如 Cubic 11 / Zpix / 賽博像素字) | 像素感 + 不受字數限制 |

設定選單「字型:原版點陣 / 現代 / 復古像素」,與「音源」並列。

## 字型選擇(TTF)

- **完整繁中**:Noto Sans/Serif CJK TC、思源黑/宋(覆蓋全,授權 OFL)。
- **復古像素 TTF**:Cubic 11、Zpix(最像原版點陣感,OFL/免費)。
- 授權:選 **OFL / 免費可散布** 的字型隨重製打包;避免版權字型。

## 實作階段(建議順序)

1. **build-time**:`decode_story_text.py --script-json` 把 35 章轉成 UTF-8 script(資料驅動)。
2. **runtime 文字層**:重製引擎做「對話框 + TTF render + 控制指令(框/頭像/翻頁)」,先跑 TTF 現代模式。
3. **頭像**:接 DATO 頭像(已可解,`01`§7),依說話者顯示 + 4 嘴型幀動畫。
4. **雙字型切換**:加原版點陣模式(atlas)與復古像素 TTF。
5. **斷行/流排**:預設保留原換行;翻譯模式做自動 word-wrap。
6. **擴充**:接 `17` 的資料驅動,新對話直接寫 UTF-8 script。

## 風險 / 注意
- **特殊符號**:機器人代號 / 兩位數字模是「整體符號」(`08`),腳本裡當字串處理,別被當一般 CJK 斷開。
- **版面長度**:翻譯或換字型會改變文字寬度 → 需流排或調框大小,別硬塞爆框。
- **保留原味**:原版點陣模式要能 1:1 還原(連同 320×200 觀感),作為「原汁」選項。
- **效能**:TTF 即時 render 可快取字形 texture(per glyph cache),避免每幀重算。
