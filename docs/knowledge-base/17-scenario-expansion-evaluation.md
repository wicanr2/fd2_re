# 17 — 擴充劇本 / 玩法的可行性評估

> 評估「在重製版裡加新戰場、新對話、新商店等機制,讓老遊戲玩出不一樣」要怎麼做、難在哪。
> 結論先講:**因為資料格式已全破、且可雙向編碼,擴充是可行的;真正要做得順,關鍵在『把原版資料轉成
> 資料驅動的格式 + 在重製引擎裡開放擴充點』,而不是改原版 EXE。**

## 我們已經握有的「可改的料」

逆向到這一步,遊戲內容幾乎都成了可讀可改的資料(各見對應章節):

| 內容 | 在哪 | 可改程度 | 依據 |
|---|---|---|---|
| **戰場地圖** | FDFIELD(構成/控制/出場) | 地形、出場數、敵我 roster、座標、回合事件、寶箱 | `01`§8 / `03` / `tools/parse_field.py` |
| **對話 / 劇情** | FDTXT + 自製字型 | 可解可重打(Unicode↔glyph 雙向)+ 控制碼(開框/頭像/翻頁) | `08` / `09` / `14` / `encode_text.py` |
| **商店** | FD2.EXE 商店表(每章 28B) | 各章武器/道具/神秘商店品項 | `03` / `02` |
| **物品 / 法術 / 敵我單位 / 成長** | FD2.EXE 數值表 | 數值、屬性、習得、價格全可調 | `03` / `docs/data/exe_tables/` |
| **AI 行為** | FD2.EXE AI(0x15140) | 評分權重、傷害門檻、可擊殺加成 | `11` |
| **頭像 / 圖塊 / 動畫 / 音樂** | DATO/FDSHAP/FIGANI/FDMUS | 可解可換可加 | `01`§7§8 / `06` / `07` / `16` |

→ **「讀得出、改得回」已具備**;擴充的素材與資料層障礙基本掃除。

## 兩條路線

### A. 改原版資料(跑在原 EXE / DOSBox)
- **可做**:調數值、換/加敵人(在出場上限內)、改商店品項、改對話(限現有 1824 字模內的字)、換素材。
- **限制**:
  - **字型只有 1824 字**——新對話若用到字型沒有的字,要先**擴字型**(在 FDOTHER#4 加 16×16 字模 + 補 glyph_map)。
  - 結構是**定長 / 上限固定**(出場 16 組事件、寶箱 16、出場人物欄位固定);加超過上限要動 EXE 程式碼。
  - 真正「新機制」(新指令、新系統)= 要改組合語言,成本高、易壞。
- **適合**:平衡性 mod、難度調整、小幅內容增補、繁中潤稿。

### B. 在重製引擎做(SDL2 / Ebiten,**建議**)
把原版資料一次性轉成**資料驅動格式**(JSON/TOML),引擎讀資料跑;新內容 = 加資料 + 開擴充點:

- **新戰場**:地圖 = tile 陣列 + 出場/事件/勝敗條件的資料檔。我們已能把 FDFIELD 轉出
  (`parse_field.py` → metadata JSON;tile 用 FDSHAP)。新增一張圖 = 編一個地圖資料檔,引擎直接吃。
- **新對話 / 分支**:對話用 UTF-8 腳本(說話者 + 文字 + 控制指令),引擎渲染。
  我們已能 FDTXT↔UTF-8 雙向(`encode_text.py`)。要分支劇情 → 在腳本格式加「條件 / 跳轉」欄位(原版沒有,重製可加)。
- **新商店 / 物品 / 法術**:全部資料表化(已 dump 成 JSON 結構),加一筆即新增。
- **新機制**(原版沒有的):因為引擎是自己寫的,可加「天氣、晝夜、地形破壞、技能樹、支線任務、
  新職業、難度模式」等——只要設計成資料 + 規則模組即可。
- **AI**:原版評分式 AI 已還原(`11`),可照搬;要更聰明就換評分函式 / 加策略。

> 重點:**重製引擎要從一開始就『資料驅動 + 留擴充點』**(地圖/對話/商店/單位/事件都走外部資料 + 腳本),
> 這樣加劇本 = 加資料檔,不用改引擎。原版的定長上限在重製裡自然解除。

## 建議做法(務實順序)

1. **先把原版資料全轉成資料驅動格式**(地圖、對話、商店、數值表 → JSON/腳本)——大半已具備工具。
2. **重製引擎以「資料 + 規則模組」為核心**,內建擴充點:地圖載入器、對話腳本直譯器(支援條件/分支)、
   商店/物品/單位資料表、事件 hook。
3. **保留「原版模式」**(完全照原資料跑,還原原作)+ **「擴充模式」**(載入額外戰場/劇情/平衡)。
4. **字型**:重製不受 1824 字模限制(可用完整中文字型或保留點陣風),新文字自由。
5. **內容工具**:做個簡單的地圖 / 對話編輯器(吃我們的資料格式),讓加劇本門檻降到「填表」。

## 風險 / 注意
- **保全原味**:擴充要可開關,別讓「不一樣」蓋掉「原汁原味」——兩種模式並存(對應音源也是 MT-32/SoundFont 並存)。
- **平衡**:加敵人/技能要重調數值;原版 AI 與成長曲線已 RE,可當基準。
- **著作權**:新增內容是衍生創作,原版資料仍只供自備原版者使用;公開的是引擎與工具,不是遊戲資料。

## 結論
**可行,且資料層障礙已大致清除。** 最划算的是「在重製引擎做資料驅動擴充」:原版資料一次轉檔,
之後加戰場 / 對話 / 商店 / 機制都只是加資料與規則模組。原版 EXE 改動僅適合小幅 mod。
建議重製從第一天就採「資料驅動 + 原版/擴充雙模式」架構。
