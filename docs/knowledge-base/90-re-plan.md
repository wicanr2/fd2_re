## 近期里程碑

### 第 2 輪(下一輪)
1. **EXE 表 dump**：用本檔 §B 錨定特徵在當前 `FD2.EXE` grep 定位，寫 `tools/dump_exe_tables.py`，
   把物品(23B)、法術(7B)、敵我單位(10B)、升級成長(11B)等整批轉成 JSON/CSV，並與攻略數值對照驗證。
2. **圖像解壓**：還原 `TITLE/BG/FDSHAP` 的壓縮(疑 RLE)，寫 `tools/decode_image.py` 輸出 PNG，
   先把標題圖、戰鬥背景、24×24 圖塊集跑出來(即時可視 = 強驗證信號)。
3. **文本解碼**：確認 `FDTXT` 字串是否 Big5，建立可讀的章節文本傾印(供中文化與劇情核對)。

### 第 3 輪以後
- 地圖格式完整解析(FDFIELD 三段對齊)→ 渲染第一張可視地圖。
- 反組譯戰鬥/命中/傷害公式，與攻略 §公式交叉驗證。
- 選定首個重製技術棧做「讀真資料 → 畫面」垂直切片。

## 硬性護欄(承接 retro-game-remake 踩雷 + dq3 教訓)

1. **引擎與版權資料分離**：原版 `FD2.EXE` / `*.DAT` / 美術 / 音樂一律 `.gitignore`，公開包只給引擎 + 工具，玩家自備合法原版。
2. **debug hook 會遮真 bug**：可破關回歸不可只靠瞬移/發道具;**另驗無 debug 的正常玩家路徑**(戰棋為逐關可達性 + 勝利條件)。
3. **打包帶「全部」資料**，不要只帶 demo 子集。
4. **不要打包測試存檔**;headless 預設不寫回玩家存檔。
5. **全程 Docker build**;Python 用 docker uv.venv，不污染系統。
6. **靜態反追溯源**：斷言「某欄位/某 offset/某規則」前，先在實檔 grep / 解碼確認對「當前版本」成立(攻略有新舊兩版 offset)。
7. **每輪更新知識庫並 push**：RE 發現 + 反思都寫進 `docs/knowledge-base/`，錯誤知識主動刪除。
