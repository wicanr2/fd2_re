## 10. 走位公式靜態反組譯(2026-07-05,回應使用者「要公式才能完整還原機制」)

追「索爾走到王座前 (8,8) 的目標/機制從哪來」,靜態反組譯繪製鏈,找到**動畫公式的一半**(修正 doc54):

### 已解:pose=方向、tick=次格滑動(繪製公式,0x127e0)
`0x127e0`(每次重繪逐單位呼叫,心跳 0x11cac→0x127a9→0x127e0)畫單位在
**`格子座標 + tick(+4) × f(pose)`**,其中 f(pose) 是方向向量(0x12891-0x128b7):
| pose(+3) | ecx=f(pose) | 方向 |
|---|---|---|
| 0 | +0x720 | **下** |
| 1 | −4 | **左** |
| 2 | −0x720(−1824) | **上** |
| 3(其他) | +4 | **右** |

⟹ **pose(+3) = 移動方向枚舉(0下1左2上3右,同 Dir/doc10 方向)**;**tick(+4) = 該步次格滑動進度**
(畫在格與相鄰格之間平滑內插,tick 遞減→滑到定格)。
**修正 doc54**:pose-oracle 判「pose≠方向」是因只看**淨格位**(草地單位 pose 循環=面向抖動、淨零位移),
沒解析到次格滑動;本輪反組譯繪製碼證實 pose 確實=方向。0x13185 序章設 Sol pose=2(上)吻合他往上走。
其他:`0x1297d`=全域 3 幀走動相位(BIOS timer 0x46c 驅動,腿部擺動);`+0x26`旗標影響繪製偏移(0x128e7)。

## §11 已解:走位控制器=0x13185「往上走一步」+ 計數迴圈(**無目標座標**)

追「索爾走到王座前的目標從哪來」,最終定位在 **`0x13185`**(序章 handler `push 2; call 0x13185`,
arg=unit index 2=索爾)。反組譯本體證實它是「**把指定單位往上移動一格**」的原語:

| 位址 | 指令 | 語意 |
|---|---|---|
| 0x1319c | `mov esi,[esp+0x20]`;`*0x50`+`[0x3a45]` | 取 unit index → &unit(0x50/單位) |
| 0x131ba | **`mov byte [esi+3], 2`** | **pose(+3)=2(上)寫死** ← 使用者問的「給 pose 叫索爾往上走的地方」 |
| 0x132fd | `mov byte [esi+1], al`(前面 `dec`) | **unit.Y(+1) 遞減 1**(實際往上移一格) |
| 0x1328e | `mov byte [esi+4], al` | tick(+4)=次格滑動時長(配 §10 繪製公式做平滑) |
| 0x13304 | `add [0x3aad], eax` | **同步捲動攝影機 Y**(camY,見 0x135dd PAN) |

**呼叫端(handler)只用兩個硬編碼計數迴圈驅動它**:
```
0x3234b xor bl,bl               ; 迴圈1
0x3234f push 2; call 0x13185    ;   往上走一步
0x03235e cmp eax,0xf; jl 0x3234f;  → 15 步
... beat ...
0x32398 push 2; call 0x13185    ; 迴圈2 → cmp eax,0xd → 13 步
```
全 handler(至 0x32974 ret)**僅此 2 個 0x13185 迴圈,共 15+13=28 步**;其後全是 ACT/PAN 對話鏡頭,無更多走位。

### 結論:走位是命令式計數步進,對話插在兩段之間
走位不是「走到某目標格」而是**往上走 N 步的計數迴圈**。handler 真實 call 序列(0x15f84=對話文字):
```
LOADCH → PAN → ACT → STEP×15(迴圈1) → ★對話#0(line0 索爾晉見父王) → STEP×13(迴圈2) → ★對話#1(line1+ 父王) → …更多對話
```
⟹ **兩段走位之間只有 1 句對話(line0)**,其餘 line1-18 都在第二段走位後。
