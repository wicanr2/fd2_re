# 20 — 第一性原理:重製可行性再確認

> 設計完成後,用第一性原理把「能不能做出可玩的 FD2 重製」拆到不可再分的事實逐一確認。
> 結論先講:**所有不可再分的必要零件都已逆向完成並實證過,重製從『研究未知』降為『工程整合』——可行。**

## 拆解:一款可玩的 FD2,最少需要哪些不可再分的能力?

把「重製一款戰棋 RPG」拆到原子需求,逐項對照我們的 RE 成果:

| # | 不可再分的必要能力 | 需要什麼 | 我們有嗎 | 依據 |
|---|---|---|---|---|
| 1 | **畫出世界** | 地圖 + 圖塊 + 調色盤 | ✅ 全 33 圖渲染、288/圖塊、VGA 調色盤 | `01`§2§8 |
| 2 | **畫角色/單位/動畫** | sprite + 戰鬥動畫 + 頭像 | ✅ FIGANI 2118 幀、DATO 136 頭像、FDSHAP sprite | `06`/`01`§7 |
| 3 | **顯示文字/劇情** | 字型 + 文本 + 對話框機制 | ✅ glyph→UTF-8 全表、控制碼、頭像對話框 | `08`/`09`/`14` |
| 4 | **播放音樂/音效** | 音源 + 曲目 | ✅ XMI→MIDI→MT-32 OGG(15 首) | `07`/`12`/`16` |

→ **9 項全部「有」**。沒有任何一項卡在「不知道怎麼做」。

## 為什麼這代表「可行」(第一性論證)

1. **資料完備性**:重製的本質是「用引擎重新表現原版的資料與規則」。原版的資料(圖/動畫/音/文字/數值/地圖)
   已**全部解成可讀格式並實證渲染/播放**;規則(AI/選單/戰鬥/成長)已從 EXE 反組譯出依據。
   → 沒有「資料黑箱」擋路。
2. **可驗證性**:每個格式都用「強約束 + 視覺/聽覺/數值對照」驗證過(解碼必為 W×H、地圖渲染正確、
   數值對攻略字面值、MT-32 實際 render)。→ 不是紙上推論,是已重現的事實。
3. **引擎存在性證明**:同型別的跨平台 SRPG/策略遊戲已用 Go/Ebiten 做成(魔法大帝重製,桌面/Web/Android)。
   → 引擎能力不是未知數。
4. **降維**:把「線性 33 關」抽象成「節點圖 + 資料驅動」(`19`),使「擴充/分支」也落在已知工程範圍。

→ 結論:**剩下的是把已知零件組起來的工程整合,不是研究突破。可行。**

## 殘留風險與處置(誠實揭露)

| 風險 | 嚴重度 | 處置 |
|---|---|---|
| **戰鬥公式細節**(命中/暴擊/傷害的確切捨入) | 低 | 有攻略公式 + EXE 表;以 headless 回歸對齊,逐步校準 |
| **動畫播放時序**(每幀停留、迴圈點) | 低 | 有全部幀;時序用合理預設,再對照原版微調 |
| **少數控制碼**(0xFFFA/FB/FC 顏色等) | 很低 | 主要碼已解;罕見碼遇到再補 |
| **FD2.SAV 存檔相容** | 不影響 | 強加密;重製用自有存檔格式(`19` 節點+旗標),不需相容 |
| **逐關平衡 / 觸發事件細節** | 低 | metadata 已有回合事件/寶箱;事件語意逐關對照補 |

→ 全是「微調/補完」級,**無「做不出來」級的阻礙**。

## 最小可玩垂直切片(MVP)的不可再分需求

要證明引擎可行,第一個垂直切片只需:**載入序章地圖 → 部署 4 名我方 → 對 AI 敵人走位/攻擊 → 達成勝利 → 播一段對話**。
所需零件 1–7 全已具備(地圖/單位/AI/選單/文字/音樂),9(引擎)即本輪要搭。→ MVP 無缺口。

## 結論

從第一性原理看,**重製的每一塊地基都已驗證存在**:資料解得開、規則查得到、引擎做得出、流程設計得了。
專案已從「逆向研究(有未知)」進入「**工程整合(無未知,只有工作量)**」階段。**可行,可開工。**
