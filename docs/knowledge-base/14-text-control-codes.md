# 14 — 文本控制碼與對話框機制

> 《炎龍騎士團2》的劇情文字怎麼被引擎一邊讀一邊畫成對話框:開框、載頭像、換行、翻頁、收框。
> 第 5 輪反組譯文本渲染器(`FD2.EXE` `0x16D00–0x17200`)還原。文字/字型格式見 `08`、`09`。

## 一句話總覽

文本是一串 **uint16**(見 `08`):多數是字模索引(畫字),少數是**控制碼**(`0xFF00`–`0xFFFF`)指揮對話框。
渲染器逐一讀 uint16,**用 sign-extend int8 比較**控制碼(讀成負值):字模就畫,控制碼就執行框 / 行 / 頁動作。

## 控制碼一覽

| 碼 | int8 | 後接 | 作用 |
|---|---|---|---|
| `0xFFEF` | −17 | 肖像 ID | **開對話框 + 載說話者頭像**(從 `DATO.DAT` 取肖像繪到框邊) |
| `0xFFEE` | −18 | 肖像 ID | **開對話框**(位置 / 樣式變體之一) |
| `0xFFED` | −19 | 肖像 ID | **開對話框**(變體) |
| `0xFFEC` | −20 | 肖像 ID | **開對話框**(變體,罕用) |
| `0xFFFE` | −2 | 字模 | **句內換行**(軟換行) |
| `0xFFFD` | −3 | 字模 | **換行 + 等待按鍵翻頁** |
| `0xFFFC` | −4 | — | 特殊繪製(罕用) |
| `0xFFFB` / `0xFFFA` | −5 / −6 | — | 罕見特效(顏色 / 其他,待細究) |
| `0xFFFF` | −1 | — | **字串結束** |

> 一句典型對白 = `[開框碼][肖像ID][『][字模…(夾雜換行碼)…][』]`,以 `0xFFFF` 收尾。
> 開框碼有 4 種(`0xFFEC`–`0xFFEF`),差在框的位置與是否顯示頭像。

## 渲染器運作(反組譯)

主迴圈(`0x1710D` 一帶):
```
讀 word → movsx eax(sign-extend)
  eax == -1 (0xFFFF) → 結束
  eax == -2 (0xFFFE) → 換行(見下)
  eax == -3 (0xFFFD) → 換行 + 等鍵翻頁
  eax == -17..-20    → 開對話框(見下)
  否則               → 當字模索引,畫到目前游標位置,游標前進
```

### 開對話框(`0xFFEF` 為例,`0x16F40`)
1. 若已有開著的框 → 先收掉(`0x17359`/`0x17A57`)。
2. 設框位置 `[0x3C67]`(`0x728`=上框 / `0x9017`=下框)。
3. 讀**下一個 word = 說話者肖像 ID**(`[esi+2]`)。
4. (`0xFFEF` 限定)以肖像 ID 從 **`DATO.DAT`** 載入該角色頭像(`call 0x11FBA`,檔名字串 linear `0x1A70`),
   繪到框邊(`0x4F6AF` blit 到 VGA `0xA0000`)。→ **這條確認了 `DATO.DAT` = 人物頭像**。
5. 設文字起點(`0xA0B4F` 上框 / `0xA951F` 下框),之後字模就畫進框內。

### 換行 / 捲動 / 翻頁
- **`0xFFFE` 換行**(`0x17119`):行計數 `[esp+0x1C]`+1,游標 = 框基底 + 行高 × 行數。
  **每框最多 3 行**;到第 3 行再換行 → `call 0x17C24`(捲動 / 等一下)後行數 −1。
- **`0xFFFD` 翻頁**(`0x16DC4`):同換行,但額外 `call 0x17A57(1)`(等待按鍵 + 清框),進下一頁。

## 相關全域

| 位址 | 意義 |
|---|---|
| `[0x3C67]` | 對話框位置(`0x728` 上 / `0x9017` 下) |
| `[0x3A85]` | 目前頭像資料緩衝 |
| `[0x3C1B]` | 目前說話者結構 |
| `0x1A70` | 字串 `"DATO.DAT"`(頭像來源) |

## 對中文化 / 重製的意義

- **改寫文字長度**要守規則:保留開框碼 + 其後的肖像 ID;每框 3 行的排版上限;`0xFFFD` 是「按鍵翻頁」點不可亂動。
- 重製對話系統:state machine = 開框(載頭像)→ 逐字 → 軟換行(3 行捲動)→ 翻頁等鍵 → 收框。
- 頭像由肖像 ID 對映角色(見 `03` 附錄 / `09`),從 `DATO.DAT` 取圖。

## 待辦
- `0xFFFC`/`0xFFFB`/`0xFFFA` 罕見碼的確切效果(疑顏色 / 名稱插入 / 特效)。
- 4 種開框碼(`0xFFEC`–`0xFFEF`)的視覺差異(框位置 / 大小 / 有無頭像)逐一截圖對照。
