# 14 — 文本控制碼與對話框機制

> 《炎龍騎士團2》的劇情文字怎麼被引擎一邊讀一邊畫成對話框:開框、載頭像、換行、翻頁、收框。
> 第 5 輪反組譯文本渲染器(`FD2.EXE` `0x16D00–0x17200`)還原。文字/字型格式見 `08`、`09`。

## 一句話總覽

文本是一串 **uint16**(見 `08`):多數是字模索引(畫字),少數是**控制碼**(`0xFF00`–`0xFFFF`)指揮對話框。
渲染器逐一讀 uint16,**用 sign-extend int8 比較**控制碼(讀成負值):字模就畫,控制碼就執行框 / 行 / 頁動作。

## 控制碼一覽

| 碼 | int8 | 後接 | 作用 |
|---|---|---|---|
| `0xFFEF` | −17 | 肖像 ID | **開對話框 + 載說話者頭像**(從 `DATO.DAT` 取肖像繪到框邊) |
| `0xFFEE` | −18 | 肖像 ID | **開對話框**(位置 / 樣式變體之一) |
| `0xFFED` | −19 | 肖像 ID | **開對話框**(變體) |
| `0xFFEC` | −20 | 肖像 ID | **開對話框**(變體,罕用) |
| `0xFFFE` | −2 | 字模 | **句內換行**(軟換行) |
| `0xFFFD` | −3 | 字模 | **換行 + 等待按鍵翻頁** |
| `0xFFFC` | −4 | — | **插入角色名**(主角名,從 `[0x53ad9]`;`0x16050` 處理) |
| `0xFFFB` | −5 | — | **插入角色名**(第二名字,從 `[0x53add]`;`0x16086`) |
| `0xFFFA` | −6 | — | **插入字串 / 數值**(`[0x53ae1]`,經 `0x377d9`/`0x37805` 格式化;`0x160a5`) |
| `0xFFFF` | −1 | — | **字串結束** |

> ⚠ 修正(第 6 輪 RE):`0xFFFC`–`0xFFFA` **不是「特效」**,是**名字 / 變數插入碼**——把全域字串指標
> `[0x53ad9]`/`[0x53add]`/`[0x53ae1]` 的內容遞迴丟回渲染器(`call 0x15f84`,共用同一畫字迴圈)畫進框內。
> 用於「○○說:…」這種把玩家命名的主角名嵌進對白的場合。

> 一句典型對白 = `[開框碼][肖像ID][『][字模…(夾雜換行碼)…][』]`,以 `0xFFFF` 收尾。
> 開框碼有 4 種(`0xFFEC`–`0xFFEF`),差在框的位置與是否顯示頭像。

## 渲染器運作(反組譯)

**文本渲染主迴圈在 `0x15F84`**(非 doc 舊稱的 `0x1710D`/`0x16D00`;`0x16D00` 是「對話播放外層 + 嘴型狀態機」,
`0x15F84` 才是逐 word 解控制碼 + 畫字的核心)。迴圈尾 `0x1630D` 讀下一 word:
```
0x1630D  movsx eax, word[esi]      ; sign-extend 讀控制碼 / 字模
  eax == -1  (0xFFFF) → 結束 (0x164ac)
  eax == -2  (0xFFFE) → 換行 (0x16322)
  eax == -3  (0xFFFD) → 換行 + 等鍵翻頁 (0x15fc4 → call 0x16e24)
  eax == -4/-5/-6     → 插入角色名 (0x16050/86/a5,見上表)
  eax == -17..-20     → 開對話框 (0x16140/0x1622a/0x16367/0x163e3,見下)
  否則(>=0)          → 字模索引,經 0x4ea2a 畫字到游標 ebp,游標 += 0x10
```

### 開對話框:4 種開框碼(`0x16140`–`0x16459`)— ⚠ 修正 `0x16F40` 錯誤

> **舊斷言「`0xFFEF`→`0x16F40` 開框」是錯的**:`0x16F40` 反組譯為 data/錯位指令,真正函式起點是 `0x16F55`,
> 且它是**radial 指令選單 / 方向鍵輸入處理**(`call 0x177FC` 寫游標方向 `[0x53C57]`、`call 0x16559` 畫指令頭像),
> **與對話開框無關**。開框碼的真正分派在文本渲染器 `0x15F84` 內,四碼各一段:

| 碼 | int8 | 處理位址 | 框位置 `[0x3C67]` | 肖像來源 | 肖像 blit | 文字起點 |
|---|---|---|---|---|---|---|
| `0xFFEF` | −17 | `0x16140` | `0x728`(上框) | DATO.DAT,ID=下個 word | `0x4E8AF` | `0xA0B4F` |
| `0xFFEE` | −18 | `0x1622A` | `0x9017`(下框) | DATO.DAT,ID=下個 word | `0x4E8E1` | `0xA951F` |
| `0xFFED` | −19 | `0x16367` | `0x728`(上框) | **戰場單位** idx→`byte[unit+7]` | `0x4E8AF` | `0xA0B4F` |
| `0xFFEC` | −20 | `0x163E3` | `0x9017`(下框) | **戰場單位** idx→`byte[unit+7]` | `0x4E8E1` | `0xA951F` |

→ 4 碼 = {上框 `0x728` / 下框 `0x9017`} × {肖像 ID 直接給(`-17/-18`)/ 用戰場單位 index 查(`-19/-20`)}。
開框流程(以 `-17` `0x16140` 為例):
1. 若已有開框(`[esp+0x18]!=0`)→ 先收:`0x16559(0)`(清嘴型)+ `0x16C57(0)` + `0x16B43`(復原存底背景)。
2. 設框位置 `[0x3C67] = 0x728`(上)或 `0x9017`(下)。
3. 讀肖像來源(`-17/-18` 取下個 word 當 DATO ID;`-19/-20` 取單位 index → `[0x53A45]+idx*80` 的 `byte[+7]`)。
4. 從 **`DATO.DAT`**(字串 linear `0x51A70`)載肖像:`0x111BA(0x51A70, [0x53A85], id)` → `[0x53A85]`=肖像 buffer。
5. blit 肖像到 VGA `0xA0000 + [0x3C67]`(上框 `0x4E8AF` / 下框 `0x4E8E1`,width `0x140`)。
6. 設文字起點(`0xA0B4F` 上 / `0xA951F` 下),之後字模畫進框內。
   → 確認 **`DATO.DAT` = 人物頭像**;`0x4F6AF` 舊位址作廢,實際 blit 用 `0x4E8AF`/`0x4E8E1`。

### 換行 / 捲動 / 翻頁
- **`0xFFFE` 換行**(`0x17119`):行計數 `[esp+0x1C]`+1,游標 = 框基底 + 行高 × 行數。
  **每框最多 3 行**;到第 3 行再換行 → `call 0x17C24`(捲動 / 等一下)後行數 −1。
- **`0xFFFD` 翻頁**(`0x16DC4`):同換行,但額外 `call 0x17A57(1)`(等待按鍵 + 清框),進下一頁。

## 對話框「框」= FDOTHER.DAT 素材圖,不是程式畫的矩形(第 6 輪 RE)

**問題**:地圖戰場對話框那個「深藍底 + 淺藍立體邊」的框,是程式 fillrect / 畫線畫的,還是素材圖?
**結論:是素材圖(sprite),從 `FDOTHER.DAT` 載入,帶縮放彈出 / 收合動畫貼上;全程沒有任何 rect 填充或畫線原語。**

- **框圖資源**:UI 載入函式 `0x25C63`–`0x25D19` 一次把 `FDOTHER.DAT`(字串 linear `0x51A4D`)多個 entry 解進全域:
  - **`0x111BA("FDOTHER.DAT", [0x53A81], 5)` → `[0x53A81]` = 對話框框圖(entry #5)**(`0x25CF2`)。
  - `0x111BA(…, [0x53A75], 4)` → `[0x53A75]` = 框內文字區底圖(entry #4)。
  - entry 1/2/3/0x1F → `[0x53A4D]`/`[0x53A89]`/`[0x53A6D]`/`[0x53EEC]`(其他 UI 片)。
  - 另 `0x111BA("FDTXT.DAT"@`0x51A43`, [0x53A7D], 0)` → `[0x53A7D]` = 對話字模 / 文字資料。
- **彈出動畫(開框)`0x165AC`**:用 `[0x53A81]` 框圖,以 `[0x53AB9]×[0x53ABD]`(框寬 / 高,單位 ×`0x18`=24px)為終點,
  逐格內插出一個由小放大的矩形,把框圖縮放 blit 進去(`0x16649` 讀 `[0x53A81]` header `word[+6]` 取尺寸再貼)。
  → 使用者述「框跳出來」= **這段 zoom-in**(縮放素材,不是淡入)。
- **收合動畫(關框)`0x16B43`**:反向 zoom-out,並用 `0x15E71` 把先前存底的 4 條背景 strip 貼回 VGA(復原被框蓋住的畫面)。
- **每 tick 重繪 / 嘴型 `0x16C57`**:同樣吃 `[0x53A81]` 框圖 + `[0x53A81]` 文字區,配 `0x4E893` 動畫進度推進。
- **框尺寸來源**:`[0x53AB9]`(寬)/`[0x53ABD]`(高)由某 context descriptor 的 `byte[+7]`/`byte[+8]` 填(`0x1040C`/`0x10415`),
  ×24px 得像素框大小;戰鬥 / 場景切換時被歸零重設(多處 `0x53AB9/ABD = 0`)。
- **3D 立體邊(深藍 + 淺藍)**:就畫在 FDOTHER entry #5 那張框圖的像素裡 → remake 應**直接抽 FDOTHER 框圖貼**,
  不要自己 drawRect + 描邊湊立體感(顏色 / 邊框寬度都燒在素材)。

## 嘴型動畫機制(第14輪 RE,2026-06-29)

對話播放時頭像**嘴巴會動**,反組譯 `0x16D00` 對話主迴圈鎖定:
- **`0x16559(idx)` = 把 DATO 第 idx 嘴型幀畫到對話框**(`[0x3A85]`=4 嘴型幀表,`[0x3C67]`=框位置;
  上框走 `0x4E8AF`、下框 `0x9017` 走 `0x4E8E1`)。idx **0 = 閉嘴、3 = 開嘴**(DATO_N_m0 / m3)。
- **嘴型狀態機**(0x16D56 起):開嘴態 → 切閉嘴(`0x16559(0)`)並設倒數 `ebp = (rand()%0x1e)+2`(隨機 2–31);
  閉嘴態 → `ebp--`,歸零切開嘴(`0x16559(3)`)。→ 嘴**隨機開合**(非固定),閉嘴 2–31 tick、開嘴一瞬,模擬說話。
- **tick = 每 2 個垂直更新**:`[0x46c]`(frame counter)與上次差 `>= 2` 才推進(0x16D00 開頭 `cmp 2`)。
- 翻頁等鍵在 `0x16DD0`(讀鍵盤掃描碼)。

**remake 對映**(忠實參數):每 2 frame 一 tick;閉嘴 `rand(2,31)` tick → 開嘴 1 tick → 閉…;頭像取 DATO m0(閉)/m3(開)。

## 對中文化 / 重製的意義

- **改寫文字長度**要守規則:保留開框碼 + 其後的肖像 ID;每框 3 行的排版上限;`0xFFFD` 是「按鍵翻頁」點不可亂動。
- 重製對話系統:state machine = 開框(載頭像)→ 逐字 → 軟換行(3 行捲動)→ 翻頁等鍵 → 收框。
- 頭像由肖像 ID 對映角色(見 `03` 附錄 / `09`),從 `DATO.DAT` 取圖。
