# 08 — 文本與自製字型格式紀錄

> 《炎龍騎士團2》怎麼在 DOS / VGA 上顯示中文。第 3 輪逆向工程整理。
> 1990 年代台灣遊戲沒有現成的中文系統,得自帶字型、用內部索引存文本——這份紀錄保存那套做法。

## 為什麼不是 Big5

DOS 原生不顯示中文。當年台灣遊戲的通行解法是**自帶一套點陣字型**,文本則存成
**字型字模的索引**(而非 Big5 編碼)。好處:① 不依賴外掛中文系統;② 只收錄遊戲用到的字,省空間;
③ 字型風格可自訂。代價:文本脫離字型就無法直接閱讀——這也是中文化的關鍵改寫點。

## 文本格式(FDTXT.DAT)

每個資源 = 一章 / 一組字串。結構:

```
+0   uint16 LE  次目錄:各字串 offset;字串數 = offsets[0] / 2
…
每字串:一串 uint16 LE,直到 0xFFFF
    0x0000 – 0x071F   字型字模索引(glyph id)
    0xFF00 – 0xFFFE   控制碼(格式 / 換行 / 名稱插入等)
    0xFFFF            字串結束
```

全 35 個資源合計 **1016 條字串、約 58000 字**,用到 **1824 個不同字模**(索引 0x000–0x71F)。

常見控制碼(出現次數):`0xFFFE`(3620)、`0xFFEE`(813)、`0xFFFD`(435)、`0xFFEF`(364)、`0xFFED`(231)…
推測為換行 / 等待按鍵 / 插入角色名 / 顏色等(語意待逐一對應)。

## 自製字型(FDOTHER.DAT 資源 #4)

```
1824 個字模,每字模 16×16 1bpp = 32 byte(16 列,每列 2 byte,MSB 在左)
總長 58368 = 1824 × 32
```

字模索引即文本的 glyph id。索引配置:**0–9 = 數字、10–35 = 英文字母**,
其後為標點與**漢字**(出現頻率最高的字模:`，`(逗號)、`的`、`我`,正是中文最常用字,反證對映正確)。

## 還原成果:可讀的遊戲文字

把文本的 glyph id 逐一對映字模畫出,即還原原版畫面文字。下圖為遊戲系統訊息:

![還原的遊戲對白](figures/dialogue.png)

```
要記錄戰況嗎？　戰況已被記錄了！　是嗎？那麼就不要了．
要讀取戰況嗎？　要離開戰場嗎？　那麼請休息吧！
決定要行軍嗎？　那麼全軍發進吧！　要結束本回合的行動嗎？
```

## 工具

`tools/decode_text.py`:
```bash
python3 decode_text.py font   FDOTHER_004.bin atlas.png     # 匯出字型字模表
python3 decode_text.py render FDOTHER_004.bin FDTXT_007.bin out.png  # 渲染某章文本
python3 decode_text.py dump   FDTXT_007.bin                  # 印 glyph-id 序列
```

## 對中文化 / 重製的意義

- 重製時可保留原字型(復古點陣感),或換高解析中文字型。
- 翻譯 / 校訂:先把 glyph id 序列配合字模轉成可讀文字(本工具已能),
  要做新中文化則建一張 glyph id → Unicode 對照表(下一步,可由字模 OCR 或人工建)。
- 控制碼語意需逐一還原(換行 / 名稱插入等),才能安全改寫文本長度。

## 待辦(後輪)
- 建 glyph id → Unicode 對照表(讓文本可全文檢索 / 翻譯)。
- 還原控制碼語意(`0xFFEE`/`0xFFFE`…)。
- 對應各 FDTXT 資源 ↔ 遊戲章節 / 場景。
