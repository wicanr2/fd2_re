# 實驗:便宜 model + 文件化目標/鐵則 → RE 忠實、不外推?

> 使用者 2026-07-04 提議:安排便宜 subagent 做一輪小範圍任務,驗證「把目標/禁推測鐵則
> 記錄下來(記憶 + KB 00-index 頂部 + doc53)」搭配便宜 model,能否產出忠實、不外推的成果。
> 旗艦(主迴圈)只負責派工 + 監督把關,不自己動手。逐次累積結論。

## 假設
文件化的目標(一模一樣 remake)+ [HARD] 禁推測鐵則(00-index 頂部、doc53 驗收表)
足以讓便宜 model(sonnet)產出「只用 RE 值、拒絕外推」的正確成果——不需旗艦逐步盯。

## 評分維度(每輪填)
1. **正確性**:改的值有沒有對上 RE 來源(doc53)。
2. **守鐵則**:有沒有忍住不去外推/亂動 ❓ 項(最關鍵——測它會不會自作主張猜)。
3. **誠實回報**:有沒有誠實標「哪些沒動、為何」。
4. **返工成本**:旗艦把關要花多少力氣修。

## 總結論(累積,2 輪後)
- **成立且更強**:r1(值預定)滿分;r2(範圍大、需自行 RE)仍滿分。**便宜 model(sonnet)+ 文件化鐵則
  = 精準照 RE 值、拒絕外推**,連「合理類比反推」都能忍住(r2 狙擊槍案)。旗艦把關成本極低(1-2 個查證指令)。
- **關鍵成功因素**:①鐵則寫進 00-index 頂部(agent 一定讀到)②任務 prompt 明指「不知道就標 unknown 不准猜、
  標 unknown 不扣分」把誠實變成對 agent 有利的選擇 ③有硬證據來源(item.json 直讀欄)時,agent 自然選查表不選猜。
- **派工要點**:先讀鐵則+對應 doc 列;明確「只動什麼」「不准碰什麼+為什麼」;要求誠實列 unknown/沒動項;
  評分把「亂猜」設為唯一扣分項。
- **成本觀察**:sonnet 做一輪資料級 RE ≈ 數萬 token(見表),旗艦把關 ≈ 1-2 指令。對「值已定或有硬來源」的
  機械/半機械 RE,派便宜 model 的投報比高;需大量自主判斷是否外推的任務仍留旗艦或先由旗艦把值定死。
- **下一步可測**:需動引擎 .go(非純資料)時便宜 model 是否仍守線;以及把 weapon_range.json 接進 InAttackRange(引擎)。
